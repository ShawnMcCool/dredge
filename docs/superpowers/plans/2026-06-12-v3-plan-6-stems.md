# earworm v3 — Plan 6: local stems

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Local stem separation (4-stem: vocals/drums/bass/other) with per-stem gain in the loop player — "mute the recorded bass, I play it." Demucs runs as a subprocess; the whole feature is tested hermetically against a fake separator and degrades gracefully when demucs is not installed.

**Architecture:** The engine learns to play a **stem set**: N equal-length SongBuffers mixed with per-stem gains *before* the stretcher (one stretcher, summed source). `server` gets a `StemSeparator` trait (Demucs impl + fake for tests), a background separation job reporting through a channel App drains in `tick()`, a stems cache at `~/.local/share/earworm/stems/<file_hash>/`, and `stems.*` commands. `song.open` auto-loads cached stems. UI: a stem mixer strip (slider + mute/solo per stem).

**Stem order is a fixed contract everywhere:** `["vocals", "drums", "bass", "other"]` (index 0–3).

**Spec:** `docs/superpowers/specs/2026-06-12-earworm-design.md` (v3 section)

```rust
/// One or more equal-length stems mixed with per-stem gains.
#[derive(Debug, Clone)]
pub struct StemSet {
    pub stems: Vec<std::sync::Arc<SongBuffer>>, // len >= 1; equal frames (pad on construction)
    pub gains: Vec<f32>,                        // same len, 0.0..=1.5
}

impl StemSet {
    pub fn single(buf: SongBuffer) -> Self { ... }
    /// Pads shorter stems with silence to the longest.
    pub fn new(stems: Vec<SongBuffer>) -> Self { ... }
    pub fn frames(&self) -> usize { ... }
    #[inline]
    pub fn frame(&self, pos: usize) -> (f32, f32) { /* sum stems[i][pos] * gains[i] */ }
}
```

Tests (in `buffer.rs`): `new` pads to longest (frames == max, padded region reads 0.0 via `frame`); `frame` sums with gains (two stems of constant 0.25/0.5 with gains 1.0/0.5 → 0.5); `single` wraps with gain 1.0.

`looper.rs`: `Looper` holds `StemSet` instead of `Arc<SongBuffer>`; all per-sample reads go through `set.frame(pos)`; new method `pub fn set_gain(&mut self, idx: usize, gain: f32)` (ignore out-of-range). Existing looper tests adapt mechanically (`StemSet::single(...)`); add one new test: two-stem looper where setting gain 0 on stem 1 mid-read silences its contribution.

`pipeline.rs`: `EngineCmd` gains `SetStemGain { idx: usize, gain: f32 }` (Copy stays intact); `Pipeline::new(set: StemSet)`; apply routes to looper. New test: pipeline with two sine stems, `SetStemGain{idx:0, gain:0.0}` → output RMS halves (use two equal-amplitude uncorrelated sines, e.g. 220 Hz and 333 Hz; RMS of the mix drops from √2·a/√2 … just assert RMS after ≈ RMS of one stem ±15%).

```rust
pub const STEM_NAMES: [&str; 4] = ["vocals", "drums", "bass", "other"];

pub trait StemSeparator: Send + Sync {
    /// Blocking; writes `<out_dir>/{vocals,drums,bass,other}.wav`, returns their paths in STEM_NAMES order.
    fn separate(&self, audio: &std::path::Path, out_dir: &std::path::Path)
        -> Result<Vec<std::path::PathBuf>, String>;
    fn is_available(&self) -> bool;
}

/// Runs `demucs -n htdemucs -o <tmp> <audio>` and moves
/// `<tmp>/htdemucs/<stem>/<name>.wav` files into place.
pub struct DemucsSeparator {
    pub binary: String, // default "demucs"
}

/// Test double: writes four copies of the input decoded to wav, each scaled
```

### Task 3: Server — stems commands, background job, auto-load

**Files:**
- Modify: `crates/server/src/app.rs`, `crates/server/src/bin/earwormd.rs`, desktop `main.rs`
- Test: `crates/server/tests/app_stems.rs`

- [x] **Step 1: Implement**

`App` additions:
- Constructor gains `separator: std::sync::Arc<dyn StemSeparator>` and `stems_dir: PathBuf` (default `~/.local/share/earworm/stems`, overridable for tests) — update all call sites (earwormd/desktop: `DemucsSeparator`; tests: `FakeSeparator`).
- Background job channel: `App` owns `(mpsc::Sender<Event>, mpsc::Receiver<Event>)`; `tick()` drains the receiver into its returned events. Separation thread: clones sender, runs separator, sends `stems_progress {song_id, state: "done"|"failed", error?}`.
- Commands:
  - `"stems.separate" {song_id}` → if cache dir `<stems_dir>/<file_hash>/` already has all 4 → `{state:"cached"}`; if separator unavailable → Err("demucs not installed — install with: uv tool install demucs (CUDA torch pulls ~2.5 GB)"); else spawn thread, return `{state:"running"}`; refuse double-start per song.
  - `"stems.status" {song_id}` → `{state: "none"|"running"|"cached"}`.
  - `"stems.gains" {gains: [f32;4]}` → 4 × `SetStemGain` engine commands (requires stems loaded — error otherwise).
- `song.open` change: if the song's cache dir has all 4 stems, decode all four (in STEM_NAMES order) → `StemSet::new`, `audio.load(set)`, response gains `"stems": true`; else single-buffer load as before, `"stems": false`. (Pitch/octave caveat does not apply — stems are same-length by Demucs contract; `StemSet::new` pads regardless.)

- [x] **Step 2: Tests** (`app_stems.rs`, FakeSeparator + Arc<Mutex<MockEngine>>, temp stems_dir):

2. `open_autoloads_cached_stems` — reopen the song: response has `stems: true`; mock got a `load` whose StemSet has 4 stems.
3. `gains_route_to_engine` — `stems.gains {gains:[1,1,0,1]}` → mock `sent` contains 4 SetStemGain incl. `{idx:2, gain:0.0}`.
4. `separate_unavailable_errors_helpfully` — a NeverAvailable test separator → error mentions "not installed".
