# Count-in Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a BPM-based count-in that plays metronome clicks before playback starts, leading the player into the song or loop.

**Architecture:** A pre-roll state inside the real-time `Pipeline` synthesizes clicks into the output buffer while holding the looper, then hands off seamlessly to the song. The `server::App` computes the beat interval from the song's analyzed BPM and pushes config down via a new `EngineCmd::SetCountIn`; config persists as a `count_in` setting. The Transport gets a beat-count stepper, hidden until the song has analysis.

**Tech Stack:** Rust (engine `Pipeline`, server `App`, rusqlite settings), Svelte 5 + TypeScript (stores, Transport component), vitest, cargo test.

**Scope note:** This plan is in two parts. **Part A** (Tasks 1–6) ships a complete, usable count-in for manual play; loops behave seamlessly (the "first loop" semantics) with the count-in firing only on the manual play press. **Part B** (Tasks 7–9) adds the *every-loop* mode and its `first ⇄ every` UI chip — the highest-risk work because of the looper's crossfade wrap and stretcher latency. Part A has no dead controls, so Part B can be deferred without leaving anything half-built.

- [ ] **Step 4: Add the click sample + pre-roll fill helpers**

Add these methods inside `impl Pipeline` (e.g. just after `render`):

```rust
    /// One interleaved click sample for the current pre-roll frame, scaled by
    /// the volume knob. Silent once the click envelope has decayed.
    fn click_sample(&self) -> f32 {
        if self.ci_click_age >= CLICK_LEN_FRAMES {
            return 0.0;
        }
        let t = self.ci_click_age as f64 / SAMPLE_RATE as f64;
        let f = if self.ci_accent {
            CLICK_FREQ_ACCENT
        } else {
            CLICK_FREQ_NORMAL
        };
        let env = (-CLICK_DECAY * t).exp();
        let amp = if self.ci_accent { CLICK_AMP } else { CLICK_AMP * 0.7 };
        ((2.0 * std::f64::consts::PI * f * t).sin() * env) as f32 * amp * self.volume
    }

    /// Fill `out` with count-in clicks. Returns the number of frames consumed
    /// by the pre-roll; when the pre-roll ends mid-buffer it clears `ci_active`
    /// and returns the frame index where the song should take over.
    fn render_count_in(&mut self, out: &mut [f32]) -> usize {
        let spacing = ((self.ci_beat_secs / self.rate) * SAMPLE_RATE as f64)
            .round()
            .max(1.0) as usize;
        let frames = out.len() / CHANNELS;
        for i in 0..frames {
            if self.ci_to_next == 0 {
                if self.ci_remaining > 0 {
                    self.ci_accent = self.ci_remaining == self.ci_beats; // first beat
                    self.ci_remaining -= 1;
                    self.ci_click_age = 0;
                    self.ci_to_next = spacing;
                } else {
                    // the final beat's full duration has elapsed → done
                    self.ci_active = false;
                    return i;
                }
            }
            let s = self.click_sample();
            out[i * CHANNELS] = s;
            out[i * CHANNELS + 1] = s;
            self.ci_click_age += 1;
            self.ci_to_next -= 1;
        }
        frames
    }
```

        if self.ci_active {

```rust
    fn countin_set(&mut self, p: Value) -> Result<Value, String> {
        #[derive(Deserialize)]
        struct P {
            beats: u32,
            loop_mode: String,
        }
        let p: P = from_params(p)?;
        let val = json!({ "beats": p.beats, "loop_mode": p.loop_mode });
        self.store.set_setting("count_in", &val).err_str()?;
        self.push_count_in();
        Ok(Value::Null)
    }

    /// Recompute and send the count-in config to the engine from the persisted
    /// setting and the open song's analyzed BPM. With no open song or no BPM,
    /// the count-in is forced off (`beats: 0`).
    fn push_count_in(&mut self) {
        let cfg = self
            .store
            .get_setting("count_in")
            .ok()
            .flatten()
            .unwrap_or(Value::Null);
        let beats = cfg.get("beats").and_then(|v| v.as_u64()).unwrap_or(0) as u32;
        let every_loop = cfg.get("loop_mode").and_then(|v| v.as_str()) == Some("every");
        let bpm = self
            .open_song
            .as_ref()
            .and_then(|o| self.library.get_analysis(o.song.id))
            .and_then(|a| a.bpm);
        let (beats, beat_secs) = match bpm {
            Some(b) if b > 0.0 && beats > 0 => (beats, beat_secs_from_bpm(b)),
            _ => (0, 0.5),
        };
        self.audio.send(EngineCmd::SetCountIn {
            beats,
            beat_secs,
            every_loop,
        });
    }
```

It returns `Option<Analysis>` (owned). `Analysis.bpm` is `Option<f64>` per `crates/practice/src/model.rs`. ## Part B — Every-loop mode (deferrable, higher risk)

> Part A ships count-in for manual play with seamless loops (the "first loop" semantics). Part B adds the *every loop* mode: count-in clicks before every loop repeat. The difficulty is that the looper crossfade-wraps internally (`crates/engine/src/looper.rs:118-124`) and the stretcher carries up to ~1 s of latency, so the re-count must be triggered on the **output** timeline, not the feed timeline. Implement only after Part A is verified.

In `pipeline.rs`, add fields to track the pipeline-driven loop when `ci_every_loop` and a region are active: `ci_loop_draining: bool`. The mechanism in `render`/`render_song`:

1. When `ci_every_loop` is true and `self.looper.region()` is `Some((start, end))`, in the feed loop clamp the read so it stops at `end` (do not let the looper crossfade-wrap): read `min(want, end - pos)` frames; when `pos == end`, set `ci_loop_draining = true` and stop feeding.
2. While `ci_loop_draining`, keep pulling from the stretcher until `stretch.pull` returns 0 (output has reached the loop end).

4. The existing pre-roll path then plays the clicks and hands back to the song at `start`.
